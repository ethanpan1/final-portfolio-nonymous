User Testing in Iterative Design

Wielding live user feedback to inform prototype design


Context: UI/UX Group Assignment

Task: Design hi-fi prototypes for a startup along multiple steps of evaluation

Skills practiced: Usability testing, prototyping, integrating critique

Part 1: The Initial Design

My group chose to design for Quell, a tech company in the fitness gaming market. Quell's selling point centers around a unique, harness-style gaming console with a custom combat fitness game, so we designed a desktop website for users to learn more about Quell and pre-order their own Quell System.

Our design process started with multiple sets of cursory sketches. We took the best parts of each sketch to produce an initial hi-fi prototype.

Part 2: The Critique

We presented our prototype to our section and an industry critic. After testing it out, they provided us with the following feedback.

# Feedback Solution
1 Reconsider the information architecture. What do users need to know to buy the product? We reworked the pages to have a clearer information flow. In addition, we added more content to the landing page and added a More Info page for users to learn more about the console and the packages it comes in.
2 The website seems to cater to gamers while the product is meant for both fitness enthusiasts and gamers. We included more information in the How It Works section on the fitness side of Quell to appeal to that audience (e.g., calories burned).
3 It's difficult to tell how well the UI works because of the filler text and placeholder images. We filled in all of the text and found more relevant photos, which was only difficult because the assingment disallowed us from using any official Quell photos.
4 General style issues such as multiple fonts, over-saturated colors, etc. We chose more similar fonts (one for headings, one for body text), desaturated the colors, and made other minor stylistic changes (like a custom logo).
The changes we made resulted in our final prototype.

Part 3: The User Test

I helmed my group's user test, which we hosted over UserTesting. I designed a test plan for three users to follow.

  • Introduction
    Please note: This is an interactive mockup made on prototyping software rather than a working website. Enter the given URL into the address bar. Once the prototype is open, clicking anywhere on the page will show you the clickable elements. Imagine you are looking for an alternative way of working out through gaming. Please pre-order one of the Quell packages and return to the home page.
  • Tasks
    1. Without leaving the homepage, describe what you would select first. Do not select anything yet. If you would leave without clicking anywhere, please explain why.
    2. Learn about Quell by clicking through the tabs. Find the general pre-order page.
    3. Purchase the Quell System Package. How easy is it to make a purchase?
    4. Return to the home page.
  • Post-Test Questionnaire
    1. If this were not a study, would you have quit at any point during the process of purchasing the product?
    2. Does the website entice you to buy the Quell?
    3. How would you describe the website in three words?
    4. How would you compare it with similar websites?

For the most part, the UserTesting results were in line with our expectations. Users found the interface clear and easy to use, all successfully completing the subtasks. We received positive feedback about the layout, clarity, and learnability of the site. Especially since we did not state that we are students, this feedback indicates high success. Nevertheless, there were still further changes we could make.

  • One user wanted the confirmation page to have more information beyond a simple thanks.
  • One user noted that he didnâ€™t expect the "Find Out More" button on the home page to lead to the same page as the How It Works tab.
  • We received some feedback to improve on the website's visual design to make it more high-quality and modern.
  • Some users did not interact with the About Us page, suggesting that it might not be necessary.

Major Takeaways

Design user tests to be as easy to follow as possible.
  • This included giving proper background information and delineating all of the subtasks.
Feedback from multiple sources is critical to the design process.
  • Multiple rounds of feedback greatly increased the usability of our prototype. A diversity of testers contributed to more holistic and helpful criticism.
Usability operates at multiple levels.
  • Usability principles can often be at odds with each other (e.g., concision vs. explanation). Striking a balance is of utmost importance.
